Consultation Summary

Project: LCID Landing Page Reveal

Objective: Cinematic Scroll Locking During Animation

The Challenge

Implement a reliable way to disable scroll until the whole animation loads, then allow scrolling. The goal was to protect the "Slider" reveal animation from being skipped by the visitor's manual scrolling.

The Technical Solution

We implemented a State-Driven Hybrid Lock. This approach combines React's declarative state with browser-native CSS and the Lenis Smooth Scroll API.

Key Insight: Native CSS (overflow: hidden) provides the most robust blocking, while Lenis API ensures the virtual scroller stays in sync. React State (isLocked) provides the "Source of Truth."

Code Architecture

1. State and Synchronization

const [isLocked, setIsLocked] = useState(true);

useEffect(() => {
  if (isLocked) {
    document.body.style.overflow = "hidden";
    lenis?.stop();
  } else {
    document.body.style.overflow = "auto";
    lenis?.start();
  }
}, [isLocked, lenis]);

2. Animation Trigger

// GSAP Footer Callback
onComplete: () => {
    setIsLocked(false); // The magic switch
    if (lenis) {
        lenis.scrollTo(".sticky-cards", { duration: 2 });
    }
}

Conclusion

This implementation ensures a premium, controlled experience for the user while following React best practices. The "isLocked" state can be expanded in the future for other page-level logic (like disabling nav interaction or showing a special cursor).
